# RealSense Camera Capture

This project is developed based on the official Intel RealSense SDK repository:
https://github.com/realsenseai/librealsense

It follows the SDK's `pyrealsense2` usage patterns for stream configuration,
frame capture, depth-to-color alignment, depth colorization, and depth
post-processing.

This project is scoped to RealSense camera handling: device access, stream
configuration, frame capture, depth-to-color alignment, metadata collection,
and camera sample storage. Computer vision algorithms, YOLO training, annotation
conversion, and detection logic belong in a separate vision project.

## Run

```powershell
D:\miniconda3\envs\realsense\python.exe capture_current_frame.py --mode single --save-pointcloud --output-dir E:\camera\test_20260612
```

`--output-dir` is required and should be set explicitly for each capture session.

## Output

Each capture uses one shared sample id:

```text
YYYYMMDD_HHMMSS_mmm
```

The saved files are:

```text
<sample_id>_color.png
<sample_id>_d2rgb.npy
<sample_id>_d2rgb_filtered.npy
<sample_id>_d2rgb_vis.jpg
<sample_id>_pointcloud.ply
<sample_id>_meta.json
index.csv
```

`color.png` is the RGB image for annotation or later vision projects.
`d2rgb.npy` is always the raw aligned uint16 depth matrix.
`d2rgb_filtered.npy` is saved only when `--enable-post-processing` is used.
`d2rgb_vis.jpg` is generated by the RealSense SDK colorizer for human inspection.
`pointcloud.ply` is saved through the RealSense SDK point cloud exporter when
`--save-pointcloud` is used.
`meta.json` stores device, stream, frame, and capture configuration metadata.
`index.csv` appends one row per complete sample.

## CLI

```powershell
D:\miniconda3\envs\realsense\python.exe capture_current_frame.py --help
```

Supported parameters:

```text
--output-dir
--mode {single,continuous,manual}
--duration-s
--interval-s
--color-size {1920x1080,1280x720,960x540,848x480,640x480,640x360,424x240,320x240}
--color-fps {6,15,30,60}
--depth-size {1280x720,848x480,640x480,640x360,480x270,424x240,320x240,320x180}
--depth-fps {6,15,30,60,90}
--warmup-frames
--auto-exposure {on,off,default}
--enable-post-processing
--save-pointcloud
--preview-window-name
--preview-max-size
```

Defaults:

```text
Mode: single
Color: 1920x1080 RGB8 30 FPS
Depth: 1280x720 Z16 30 FPS
Align: depth -> color
Stereo post-processing: disabled by default
Depth visualization: Dynamic, Jet, histogram equalization enabled
Warmup frames: 20
Auto exposure: on
Continuous interval: 1.0 seconds
```

Use `--enable-post-processing` to save an additional Viewer-like Stereo Module
post-processed depth matrix as `d2rgb_filtered.npy`. `d2rgb.npy` remains the raw
depth-to-color aligned matrix.

## Recommended Use

Use `color.png` and raw `d2rgb.npy` for annotation, calibration, pixel-level
lookup, training data, and downstream vision algorithms because they keep the
RGB/depth pixel correspondence intact.

Use `d2rgb_filtered.npy` and `d2rgb_vis.jpg` for visual inspection,
presentation, and rough scene understanding when a smoother depth image is more
useful than preserving every raw measurement.

Continuous capture saves repeated samples with the same file naming rules:

```powershell
D:\miniconda3\envs\realsense\python.exe capture_current_frame.py --mode continuous --duration-s 10 --save-pointcloud --output-dir E:\camera\test_20260612
```

The first sample is saved immediately after warmup, then one sample is saved
every `--interval-s` seconds. Preview opens one fixed layout window with RGB
and depth visualization on the left and an interactive 3D point cloud view on
the right. The point cloud panel follows the RealSense OpenCV viewer style with
a ground grid, camera frustum, and coordinate axes. Drag inside the point cloud
panel with the left mouse button to rotate, drag with the right mouse button to
pan, and use the mouse wheel to zoom. Press `q` or `Esc` to stop early.

Manual capture keeps the preview window open and saves one sample whenever
Space is pressed:

```powershell
D:\miniconda3\envs\realsense\python.exe capture_current_frame.py --mode manual --save-pointcloud --output-dir E:\camera\test_20260612
```

## Common Issues

If capture fails because no camera is detected, check USB connection and run
RealSense Viewer once to confirm the device is visible.

If capture fails because the device is busy, close RealSense Viewer or any
other program using the camera.
